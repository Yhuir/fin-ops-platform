# Prompt 19: 项目明细强导出后端底座

## 目标

把现有 `导出项目明细` 从单表导出升级成强导出工作簿底座，先完成后端数据模型、默认导出 payload 和多 sheet workbook 生成。

## 你要做的事

- 新增 `ProjectDetailExportService` 或等价服务
- 基于项目名称生成“项目成本导出上下文”
- 输出多 sheet workbook，而不是单表
- 保留当前 `xlsx` 导出链路兼容
- 为导出模型和 workbook 生成补测试

## 约束

- 先只做一个项目的导出
- 不做 ZIP，不打包原始附件
- 不做前端高级导出 UI
- 不破坏当前成本统计页已有导出能力
- 正常成本、异常、忽略要分表，不要混在同一个明细表

## 交付物

- `ProjectDetailExportService`
- 新的项目导出 payload 结构
- 多 sheet `xlsx` 导出能力
- 后端单元测试和 API 测试

## 推荐工作簿结构

- `导出说明`
- `项目汇总`
- `按费用类型汇总`
- `按费用内容汇总`
- `流水明细`
- `OA关联明细`
- `发票关联明细`
- `异常与未闭环`

## 验收标准

- 导出一个项目时，生成的是多 sheet workbook
- `项目汇总`、`按费用类型汇总`、`流水明细` 三张表必须存在
- 正常明细和异常明细不能混表
- 文件名包含项目名称和时间范围
- 测试通过

## 可直接使用的 Prompt

```md
请为当前系统实现“项目明细强导出”的后端底座。

背景：
- 当前成本统计页已经支持项目视图和基础导出
- 现在的“导出项目明细”只是单表导出，不够财务复核使用
- 需要把它升级成多 sheet 的项目成本工作簿

实现要求：

1. 新增独立导出服务，建议命名为 `ProjectDetailExportService`
2. 以“一个项目”为单位生成导出上下文
3. 导出结果至少包含以下 sheet：
   - 导出说明
   - 项目汇总
   - 按费用类型汇总
   - 按费用内容汇总
   - 流水明细
   - OA关联明细
   - 发票关联明细
   - 异常与未闭环
4. 正常成本、已处理异常、已忽略记录必须分开
5. 导出格式继续优先 `xlsx`
6. 补单元测试和 API 测试

注意：
- 这一步先只做后端底座和默认导出，不做前端高级导出配置。
- 不要做 ZIP。
- 不要破坏现有成本统计页其他导出。
```
