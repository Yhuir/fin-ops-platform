# Prompt 20: 成本统计导出中心交互

## 目标

把成本统计页里分散的导出入口收口成一个统一的 `导出中心`，支持 `按时间 / 按项目 / 按费用类型` 三种导出模式，并在下载前先做结构化预览。

## 你要做的事

- 删除旧的 `导出项目明细 / 高级导出 / 最近导出`
- 在成本统计页头部只保留一个 `导出中心` 按钮
- 点击后打开弹出窗口，不新开页面
- 弹窗里支持切换：
  - `按时间`
  - `按项目`
  - `按费用类型`
- 每种模式都支持自己的筛选项与 `仅预览 / 导出`

## 交互要求

### 按时间

- 支持：
  - 自定义月份
  - 自定义时间区间（精确到日）
- 预览内容至少包括：
  - 预计导出条数
  - 预计 sheet 数
  - 金额合计
  - 样例前几行

### 按项目

- 不受月份限制，默认全部期间
- 先选 `项目`
- 再多选 `费用类型`
- 费用类型支持：
  - 全选
  - 清空
  - 勾选多个

### 按费用类型

- 先多选 `费用类型`
- 再选择：
  - 单月
  - 自定义时间区间（精确到日）

## 约束

- 风格必须保持“浅色、企业后台、清晰稳重”
- 不做大屏看板
- 不做花哨 BI
- 不做多步骤向导
- 导出中心必须复用后端真实导出能力，不做前端假下载
- `仅预览` 不是 toast，必须展示真实预览结果

## 交付物

- 成本统计页统一 `导出中心` 交互
- 导出中心弹窗
- 三种模式的筛选表单
- `仅预览` 结果区
- 真实导出 loading / success / error 反馈
- 前端测试

## 验收标准

- 页面上只剩一个 `导出中心`
- 不再出现 `高级导出` 和 `最近导出`
- `按时间` 支持自定义月份和自定义时间区间（精确到日）
- `按项目` 支持项目选择和费用类型多选
- `按费用类型` 支持费用类型多选和月份/区间选择
- `仅预览` 会显示真实预览结果
- `导出` 会调用真实导出接口
- 前端测试通过
