# Prompt 24: 关联台强搜索筛选、命中高亮与 QA 收口

## 目标

把关联台强搜索从“能搜能跳”打磨成“可真正高频使用”的版本，补齐命中高亮、错误态、筛选联动和 QA 回归。

## 你要做的事

- 为结果项增加命中字段标签
- 对命中的关键词片段做局部高亮
- 完善 loading / empty / error 状态
- 打磨筛选联动：
  - scope
  - month
  - project
  - status
- 确保不同筛选组合下结果数量和分组正确
- 做回归测试和页面 QA

## 约束

- 不要新增新的搜索语法
- 不要做拼音搜索
- 不要做搜索历史
- 不要改变 Prompt 22 / 23 已确定的接口结构
- 优先把第一版的易用性和稳定性收口

## 重点验收

- 命中字段标签准确
- 命中高亮不应整行染色，只高亮片段
- 搜索失败时有明确错误态
- 搜索无结果时三组空态清晰
- 月份 / 项目 / 状态筛选组合下结果正确
- 跳转高亮不会丢失

## 可直接使用的 Prompt

```md
请对“关联台强搜索”做最终收口和 QA。

背景：
- 搜索后端接口已经完成
- 搜索弹窗和跳转定位已经完成
- 现在需要把第一版做成真正可用的版本

实现要求：

1. 为结果项增加“命中字段”标签
2. 对命中的关键词做局部高亮，不要整行染色
3. 完善 loading / empty / error 状态
4. 验证并修正以下筛选联动：
   - scope
   - month
   - project
   - status
5. 为结果分组、命中高亮、空态、错误态、跳转高亮补测试
6. 跑前后端回归测试，确保第一版稳定

注意：
- 不要新增拼音搜索和搜索历史。
- 保持接口结构稳定，以收口和 QA 为主。
```
