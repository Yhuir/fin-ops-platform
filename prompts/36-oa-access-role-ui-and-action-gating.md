# Prompt 36：访问账户管理 UI 与读写权限前端联动

## 目标

基于新的 OA 权限模型，重构关联台 `设置 -> 访问账户管理`，并把读写权限联动到前端页面行为中。

## 本次范围

- 重构 `访问账户管理` UI
- 管理员可维护：
  - 可访问账户
  - 只读导出账户
  - 全操作账户
- 非管理员隐藏这块设置
- 前端按 `read_export_only` 和 `full_access` 控制交互

## 必须完成

1. `访问账户管理` UI 不再只是一个简单白名单输入框
2. 只允许 `YNSYLP005` 看到并使用该设置
3. 非管理员即使能进入设置，也看不到这一块
4. `只可看和只可导出` 用户前端行为：
   - 可查看页面
   - 可查看详情
   - 可搜索
   - 可导出
   - 不可导入
   - 不可确认关联 / 取消关联 / 异常处理 / 忽略 / 设置保存
5. `所有操作均可` 用户保留正常业务交互
6. 补前端测试

## 验收标准

- 管理员能保存新的访问控制配置
- 只读用户不会看到或无法点击写操作入口
- 导出、详情、搜索仍然可用
- 非管理员看不到 `访问账户管理`
