# Prompt 05: 台账与提醒

## 目标

把未闭环核销事项转成财务可持续跟进的台账，并建立提醒机制。

## 你要做的事

- 实现催款台账
- 实现催票台账
- 实现退款台账
- 实现预收/预付台账
- 支持预计处理日期、状态、跟进备注
- 建立提醒任务或提醒接口

## 约束

- 台账必须来自真实业务动作，不允许纯手填孤立数据
- 台账状态变化要有审计记录
- 提醒逻辑要可重复执行且不重复轰炸

## 交付物

- 台账模型与页面或接口
- 提醒模型与调度骨架
- 台账流转规则

## 验收标准

- 部分核销自动进入催款台账
- 多付款缺票自动进入催票台账
- 无票收款 / 溢收可进入预收或退款待处理台账
- 能按预计日期查看待处理事项

## 可直接使用的 Prompt

```md
请实现“台账与提醒”模块。

先阅读：
- docs/prd.md
- docs/domain-model.md
- docs/solution-design.md

业务要求：
- 对所有未闭环事项建立台账，不再依赖线下 Excel 跟进。

实现要求：

1. 建立 FollowUpLedger 或等价模型，至少支持：
   - 催款
   - 催票
   - 退款
   - 预收
   - 预付
   - 待开销项票
   - 待付款提醒
   - 外部往来登记
   - 非税收入登记
2. 台账必须由核销结果自动生成，不要只做手工录入页面。
3. 支持预计处理日期、责任人、状态、跟进备注。
4. 建立 Reminder 或等价模型，以及最小可运行的提醒调度骨架。
5. 支持查看即将到期和已逾期事项。
6. 补充必要测试。

特别注意：
- 台账是业务闭环中心，不是附属备注表。
- 提醒机制要避免重复发送或重复创建。

输出要求：
- 直接修改代码
- 说明台账生成规则
- 说明提醒如何触发和验证
```
