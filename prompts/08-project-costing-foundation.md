# Prompt 08: 项目成本测算底座

## 目标

在不抢占核销 MVP 主线的前提下，为未来项目成本测算建立最小可用底座。

## 你要做的事

- 建立项目主数据模型
- 在发票、流水、核销单、台账上贯通项目关联字段
- 提供基础项目归集视图或查询能力
- 支持手动指定项目归属
- 保留审计记录

## 约束

- 当前阶段只做底座，不做复杂成本算法
- 不要为了项目成本测算破坏现有核销闭环
- 项目归属变更必须可追溯

## 交付物

- Project 模型
- 项目关联字段贯通
- 基础项目查询或看板接口
- 审计记录

## 验收标准

- 核心业务对象可挂项目
- 能按项目查看已核销、未核销、待跟进金额
- 后续可继续演进为正式项目成本测算模块

## 可直接使用的 Prompt

```md
请为当前系统建立“项目成本测算底座”。

先阅读：
- docs/pre.md
- docs/domain-model.md
- docs/solution-design.md
- docs/roadmap.md

背景：
当前主线仍然是 Reconciliation，但未来蓝图明确包含 Project Costing。现在先做项目维度预留和最小归集能力。

实现要求：

1. 增加 Project 或等价主数据模型。
2. 在 Invoice、BankTransaction、ReconciliationCase、FollowUpLedger 等核心对象上贯通 project_id 或等价字段。
3. 支持手动指定项目归属，并保留审计记录。
4. 提供基础查询能力，让用户能按项目查看收入、支出、未闭环事项或等价信息。
5. 当前不要实现复杂成本分摊算法，但要让后续扩展路径清晰。

注意：
- 目标是“底座”，不是一步到位做完整成本系统。
- 不要让项目维度改造破坏现有核销模型。

输出要求：
- 直接修改代码
- 说明项目维度如何贯通到现有对象
- 说明后续继续做正式成本测算时的扩展点
```
