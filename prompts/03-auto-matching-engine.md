# Prompt 03: 自动匹配引擎

## 目标

实现基于客商和金额的自动匹配与建议匹配能力。

## 你要做的事

- 建立匹配规则引擎
- 区分自动核销、建议匹配、待人工处理
- 支持标准一对一
- 支持多票一付 / 一付多票候选建议
- 支持部分核销和溢收/缺票的候选拆分

## 约束

- 自动匹配要可解释，不能是黑盒
- 不要把低置信度结果直接自动平账
- 规则命中信息要能被前端或工作台查看

## 交付物

- 匹配规则模块
- 匹配结果模型
- 匹配解释信息
- 基础测试用例

## 验收标准

- 正常一对一场景可自动匹配
- 复杂场景至少能产出建议结果
- 无法确定的情况进入人工工作台

## 可直接使用的 Prompt

```md
请在现有仓库中实现“自动匹配引擎”。

先阅读：
- docs/prd.md
- docs/domain-model.md
- docs/solution-design.md

业务目标：
- 根据客商、金额、收付款方向自动匹配发票和银行流水
- 把结果分成：自动核销、建议匹配、待人工处理

实现要求：

1. 优先支持标准一对一匹配。
2. 支持同客商下的多票一付 / 一付多票候选建议。
3. 支持部分核销、溢收款、缺票等场景的建议拆分，但低置信度结果不要直接自动闭环。
4. 为每条匹配结果记录命中的规则、置信度、解释信息。
5. 输出结果要能被人工核销工作台直接消费。
6. 补充覆盖关键场景的测试。

请特别注意：
- 自动匹配引擎只做规则执行和结果输出，不替代人工判断。
- 复杂异常应该进入人工工作台，而不是硬编码自动平账。

输出要求：
- 直接修改代码
- 说明你实现了哪些规则
- 说明每类结果如何流转到下一步
```
