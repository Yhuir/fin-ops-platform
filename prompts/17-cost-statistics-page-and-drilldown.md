# Prompt 17: 成本统计页面与下钻

## 目标

在 React 正式前端中新增 `成本统计` 页面与路由，并把月份汇总、项目明细、具体流水的下钻链路做通。

## 你要做的事

- 在 `税金抵扣` 右侧新增 `成本统计` 导航按钮
- 新增独立页面路由，不使用弹窗
- 复用全局年份 / 月份上下文
- 实现月份汇总视图
- 实现项目明细视图
- 实现具体流水详情层
- 使用面包屑或返回路径完成页面内下钻

## 约束

- 不要把主下钻流程做成弹窗
- 页面排版要清晰，以表格为主
- 当前阶段不做导出
- 不要重写现有关联台或税金页结构

## 交付物

- 新页面：`/cost-statistics`
- 新页面组件与 API 绑定
- 顶部导航按钮
- 下钻交互与基础状态

## 验收标准

- 用户可从顶部进入 `成本统计`
- 页面可按月份显示：
  - 项目名称
  - 费用类型
  - 金额
  - 费用内容
- 可点击下钻到项目明细
- 可继续下钻到具体流水
- loading / empty / error 清晰可见

## 可直接使用的 Prompt

```md
请为当前 React 前端新增“成本统计”页面。

背景：
- 顶部导航已存在 `关联台 / 导入中心 / 税金抵扣`
- 现在需要在 `税金抵扣` 右侧新增 `成本统计`
- 这是一个独立页面，不是弹窗
- 页面共享当前全局年份 / 月份上下文

实现要求：

1. 新增 `成本统计` 路由与页面。
2. 页面默认打开“按月份统计”视图，表格显示：
   - 项目名称
   - 费用类型
   - 金额
   - 费用内容
3. 用户点击某一行后，进入项目明细层，显示：
   - 时间
   - 费用类型
   - 金额
   - 费用内容
4. 用户继续点击后，进入具体流水层，显示该条具体流水及对应 OA 成本字段。
5. 页面内使用面包屑或返回路径完成下钻，不要用弹窗承载主流程。
6. 补前端测试，覆盖：
   - 路由进入
   - 年月联动
   - 月份汇总 -> 项目明细 -> 具体流水
   - empty / loading / error

注意：
- 保持页面清晰，仍以表格为主。
- 不要把这一步和导出功能绑定到一起。
```
