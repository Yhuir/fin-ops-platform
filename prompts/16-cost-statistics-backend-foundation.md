# Prompt 16: 成本统计后端底座

## 目标

为新的 `成本统计` 页面建立后端只读统计能力，基于**支出类银行流水 + 已关联 OA 成本字段**输出月份汇总、项目明细和具体流水详情。

## 你要做的事

- 新增成本统计只读服务
- 定义月份汇总读模型
- 定义项目明细读模型
- 定义具体流水详情读模型
- 提供成本统计 API
- 补后端测试

## 约束

- 统计以银行流水为主，不以发票为主
- 仅统计能够取到 `项目名称 / 费用类型 / 费用内容` 的支出流水
- 当前阶段不做复杂成本分摊、毛利计算、跨月趋势
- 不破坏现有工作台和项目归集逻辑

## 交付物

- `CostStatisticsService` 或等价服务
- 接口：
  - `GET /api/cost-statistics?month=YYYY-MM`
  - `GET /api/cost-statistics/projects/{project_name}?month=YYYY-MM`
  - `GET /api/cost-statistics/transactions/{transaction_id}`
- 对应测试

## 验收标准

- 当前月份可返回 `项目名称 / 费用类型 / 金额 / 费用内容`
- 点击项目可查询项目内明细
- 可下钻到具体银行流水及其对应 OA 成本字段
- 只读口径稳定、可解释

## 可直接使用的 Prompt

```md
请为当前系统实现“成本统计”的后端底座。

背景：
- 新页面入口名称是 `成本统计`
- 页面建立在现有 OA、银行流水、项目归集能力之上
- 成本统计以银行流水为主数据源
- 仅统计支出类、且已能取得 OA `项目名称 / 费用类型 / 费用内容` 的流水

实现要求：

1. 新增成本统计只读服务，建议命名为 `CostStatisticsService`。
2. 提供月份汇总接口：
   - 返回字段至少包含 `项目名称 / 费用类型 / 金额 / 费用内容`
3. 提供项目明细接口：
   - 返回字段至少包含 `时间 / 费用类型 / 金额 / 费用内容`
4. 提供具体流水详情接口：
   - 返回具体银行流水和对应 OA 成本字段
5. 明确排除规则：
   - 收入流水
   - 未确认关联流水
   - 缺少 OA 成本字段的记录
   - 已忽略记录
6. 为上述接口补单元测试和 API 测试。

注意：
- 这一步只做后端统计底座，不做前端页面，不做导出。
- 不要把“成本统计”做成新的核销流程。

输出要求：
- 直接改代码
- 说明入统口径
- 说明 API 返回结构
- 说明哪些记录被排除
```
