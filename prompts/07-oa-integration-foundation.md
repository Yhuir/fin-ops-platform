# Prompt 07: OA 集成底座

## 目标

建立 OA 接入所需的适配层、主数据映射和同步骨架，但不在这一阶段做重业务耦合。

## 你要做的事

- 抽象集成适配器接口
- 设计 OA 外部 ID 映射
- 预留审批单、付款申请、报销单、项目主数据同步模型
- 实现一个 Mock OA 适配器
- 实现基础同步任务或同步接口

## 约束

- OA 接入不能直接侵入核心核销逻辑
- 映射关系要可审计、可重试
- 暂不强求和真实 OA 打通

## 交付物

- Integration Hub 模块
- OA 映射表或模型
- Mock 适配器
- 同步任务骨架

## 验收标准

- 系统已有稳定的 OA 接入边界
- 核销对象可关联到未来的 OA 单据或项目
- 后续对接真实 OA 时不需要重构核心核销模块

## 可直接使用的 Prompt

```md
请为当前系统建立“OA 集成底座”。

先阅读：
- docs/pre.md
- docs/domain-model.md
- docs/solution-design.md
- docs/roadmap.md

背景：
当前切入点仍然是银企核销，但未来蓝图明确包含 OA 系统接入。现在先做边界和底座，不要求打通真实 OA。

实现要求：

1. 抽象 Integration Hub 或等价模块，避免 OA 逻辑直接侵入核销核心模块。
2. 为客商、项目、审批单、付款申请、报销单预留 external_id / oa_id 映射能力。
3. 设计并实现一个 Mock OA 适配器，模拟拉取主数据和单据。
4. 提供同步任务骨架、失败重试入口或同步接口。
5. 让核心业务对象未来可以挂接 OA 单据，但当前不要强耦合。

输出要求：
- 直接修改代码
- 说明集成边界如何划分
- 说明真实 OA 接入时最小改动点在哪里
```
