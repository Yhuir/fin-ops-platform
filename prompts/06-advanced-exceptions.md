# Prompt 06: 高阶异常处理

## 目标

补齐差额核销、红字发票、内部抵扣等复杂财务场景。

## 你要做的事

- 支持差额核销
- 支持手续费/抹零等差额原因分类
- 支持负数发票与反向流水核销
- 支持跨应收/应付板块内部抵扣
- 补充复杂场景测试

## 约束

- 复杂异常必须保留依据和备注
- 差额原因必须结构化记录
- 内部抵扣不能伪装成真实银行流水

## 交付物

- 差额核销流程
- 红字发票处理流程
- OffsetNote 或等价模型
- 覆盖复杂场景的测试

## 验收标准

- H/I/J 场景可闭环
- 审计记录完整
- 不破坏既有标准核销流程

## 可直接使用的 Prompt

```md
请在当前系统中实现“高阶异常处理”。

先阅读：
- docs/prd.md
- docs/domain-model.md
- docs/solution-design.md

需要覆盖的业务：
- 差额核销（手续费、抹零）
- 红字发票与退款
- 债务互抵 / 跨板块内部抵扣

实现要求：

1. 支持差额核销，并记录 difference_amount 和 difference_reason。
2. 支持负数发票参与核销，兼容反向银行流水。
3. 支持内部抵扣单 OffsetNote 或等价模型，不依赖真实银行流水即可完成跨板块平账。
4. 所有复杂异常都要保留备注、操作人、时间、关联对象。
5. 补充覆盖 H/I/J 场景的测试或验收样例。

注意：
- 内部抵扣是独立业务对象，不要伪造银行流水。
- 差额原因要结构化，不要只存自由文本。

输出要求：
- 直接修改代码
- 说明三个异常能力分别如何实现
- 说明对原有流程的影响
```
