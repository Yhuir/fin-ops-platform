# 关联工作台 V2 开发概览

## 1. 目标

本轮开发目标不是继续做静态原型，而是把已经确认的交互落成一套可演进的产品骨架，覆盖两个页面：

1. `OA & 银行流水 & 进销项发票关联台`
2. `销项票税金 - 进项票税金`

核心结果：

- 工作台支持按月份切换数据
- 三栏支持完整拖拽，能够收起到 0 宽
- 点击行只选中，不自动弹详情
- 详情只能通过行内 `详情` 按钮打开右侧抽屉
- 银行栏提供更丰富的行内动作
- 税金抵扣页支持按月勾选发票并实时计算抵扣结果

## 2. 页面边界

### 2.1 主工作台

主工作台由以下区域组成：

- 顶部栏：标题、月份选择、税金抵扣入口
- 筛选栏：搜索、状态筛选、银行筛选、栏位显隐按钮
- 概览指标：OA 数、流水数、发票数、已配对数、未配对数、异常数
- 已配对区：较矮的浏览区
- 未配对区：较高的主操作区
- 右侧详情抽屉：按需打开

### 2.2 税金抵扣页

税金抵扣页由以下区域组成：

- 顶部栏：月份选择、返回主工作台
- 汇总卡片：销项税额、进项税额、抵扣额、应纳或留抵结果
- 左中双表：销项票税金清单、进项票税金清单
- 右侧结果区：公式、计算结果、生成结果动作

## 3. 最终确认的交互口径

以下口径已经确定，开发过程中不能回退：

- 三栏中间两条竖线必须支持完整拖拽，不只是局部微调
- 任意一栏都可以被拖到 0 宽，视为临时收起
- 点击表格行只做选中和同组高亮，不自动打开详情
- 详情抽屉只能通过每行的 `详情` 按钮打开
- 工作台主要动作尽量行内完成，不设置底部固定操作条
- 银行栏的“关联情况 / 取消关联 / 异常处理”属于主表直接操作
- 详情字段统一进抽屉，不做子行展开

## 4. 推荐实现分期

### 第一阶段：前端工作台骨架

- 建立前端工程
- 路由与页面壳体
- 月份上下文
- 三栏二维工作台
- 详情抽屉壳体

### 第二阶段：交互完成度

- 行选中与同组联动
- 完整拖拽与收起
- 行内操作按钮与状态标签
- 税金抵扣页交互

### 第三阶段：后端与真实数据

- 主工作台接口
- 税金抵扣接口
- OA 适配器接口
- 银行账户识别
- 详情字段返回

### 第四阶段：验证与交付

- 前端单元测试
- 后端接口测试
- 端到端关键路径验收
- 文档和 prompt 对齐

## 5. 技术决策

为了让这批任务可执行，建议这轮正式实现使用：

- 前端：`Vite + React + TypeScript`
- 前端测试：`Vitest + Testing Library`
- 后端：继续沿用当前仓库的 Python 服务
- 接口风格：先使用 JSON HTTP API
- 状态管理：优先使用 React state/context，小范围状态不引入重型状态库

不建议在第一轮引入重量级表格库。当前需求的关键难点是：

- 三栏联动
- 行内动作
- 详情抽屉
- 分隔线拖拽

这些用标准表格 + CSS + React 状态即可完成。
