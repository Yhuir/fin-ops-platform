# 成本统计“导出项目明细”强功能设计

日期：2026-04-01

## 目标

把当前“导出项目明细”从简单表格导出，升级成一个真正可供财务、项目经理、审计复核使用的强导出能力。

它的目标不是只把当前表格导出成一个 sheet，而是导出一个**结构完整、字段可追溯、适合复核与归档的项目成本工作簿**。

## 背景

当前系统已经具备：

- 成本统计页三种视图：
  - 按时间
  - 按项目
  - 按费用类型
- 基于已配对 `OA + 支出流水` 的成本统计口径
- 当前层级导出能力
- `xlsx` 导出链路

当前问题是：

- `导出项目明细` 只是单张表
- 没有费用类型汇总层
- 没有 OA / 发票 / 异常链路说明
- 没有导出配置能力
- 不能作为正式“项目成本包”使用

## 范围

本轮设计覆盖：

- 项目明细强导出工作簿结构
- 导出配置与交互
- 后端导出任务模型
- 导出历史与可追溯性
- 导出验收与 QA 范围

本轮不做：

- PDF 成本报告
- Word 报告排版
- 图表型 BI 大盘导出
- 原始附件二进制打包 ZIP
- 定时自动发送邮件

## 设计原则

### 1. 导出不是截图，而是工作簿

导出结果必须是一个适合继续查账、筛选、复核的 Excel 工作簿，而不是简单复制当前页面。

### 2. 项目明细导出以“项目”为核心

导出对象是“一个项目的一份成本包”，而不是某一个当前表格切片。

### 3. 既能汇总，也能追溯到流水

导出结果必须同时满足：

- 财务看汇总
- 项目经理看费用结构
- 审计看来源链路

### 4. 当前页面选择应成为导出默认值，而不是唯一值

用户在页面里选中了某个项目时，应能一键导出；但也应该有高级配置能力，决定是否带异常、是否带发票、是否导出全部期间等。

## 用户场景

### 场景 1：财务月末归档

财务在成本统计页按项目选中某个项目，导出该项目本月或全部期间的成本工作簿，用于归档和复核。

### 场景 2：项目负责人查看成本构成

项目负责人按项目查看某个项目，导出按费用类型分类后的成本明细，查看大头费用在哪些类型。

### 场景 3：审计或管理层复核

复核人员希望看到：

- 汇总金额
- 费用类型占比
- 具体流水明细
- 对应 OA 成本字段
- 是否存在异常 / 忽略 / 未闭环

## 功能方案

推荐方案：`多 Sheet 强工作簿 + 高级导出配置 + 导出历史`

不推荐把强导出继续做成单 sheet 平铺，因为那样无法承接真实财务复核场景。

## 工作簿结构

导出一个项目时，建议生成以下 sheet：

### 1. 导出说明

字段：

- 项目名称
- 导出时间
- 导出人
- 导出范围
- 时间范围
- 导出视图来源
- 当前筛选条件
- 数据口径说明

用途：

- 让导出的文件脱离系统后仍可解释来源

### 2. 项目汇总

字段：

- 项目名称
- 统计期间
- 总支出金额
- 支出流水笔数
- 费用类型数
- 已关联 OA 笔数
- 已关联发票笔数
- 已处理异常笔数
- 已忽略笔数

用途：

- 给管理层和财务先看总览

### 3. 按费用类型汇总

字段：

- 费用类型
- 金额
- 占比
- 笔数
- 典型费用内容

用途：

- 看项目成本结构

### 4. 按费用内容汇总

字段：

- 费用类型
- 费用内容
- 金额
- 笔数

用途：

- 看同一费用类型下，具体花在什么内容上

### 5. 流水明细

字段：

- 交易时间
- 交易流水 ID
- 对方户名
- 支付账户
- 金额
- 摘要 / 备注
- 项目名称
- 费用类型
- 费用内容
- 关联状态
- OA 单号
- 发票号码

用途：

- 真正的查账工作表

### 6. OA 关联明细

字段：

- OA 单号
- 申请人
- 项目名称
- 费用类型
- 费用内容
- OA 金额
- 对应流水 ID
- 关联状态

用途：

- 看成本语义来自哪条 OA

### 7. 发票关联明细

字段：

- 发票号码
- 销方名称
- 购方名称
- 发票金额
- 税额
- 对应流水 ID
- 对应项目名称
- 关联状态

用途：

- 在项目维度审查票流一致性

### 8. 异常与未闭环

字段：

- 记录类型
- 记录 ID
- 项目名称
- 费用类型
- 金额
- 当前状态
- 异常标签
- 备注

用途：

- 把不干净的记录单独拿出来，不混在正常成本明细里

## 导出模式

### 快速导出

入口：

- 成本统计页 `按项目` 视图
- 用户已经选中某个项目
- 点击 `导出项目明细`

行为：

- 直接按默认配置导出
- 默认配置为：
  - 导出全部期间
  - 包含项目汇总
  - 包含费用类型汇总
  - 包含流水明细
  - 包含 OA 关联明细
  - 包含发票关联明细
  - 包含异常与未闭环

### 高级导出

入口：

- `导出项目明细` 右侧下拉或二级按钮

可配置项：

- 时间范围：
  - 当前月份
  - 全部期间
  - 自定义起止月份
- 内容范围：
  - 仅已配对
  - 包含已处理异常
  - 包含已忽略
  - 包含发票明细
  - 包含 OA 明细
  - 包含费用内容汇总
- 排序方式：
  - 按时间
  - 按费用类型
  - 按金额降序

## 前端交互设计

### 入口

只在 `按项目` 视图增强导出。

原因：

- 这是“项目明细导出”
- 不是整个成本统计页的泛导出升级

### 交互结构

- 顶部保留 `导出项目明细`
- 默认点击快速导出
- 旁边提供 `高级导出`
- 导出期间显示阻塞式 loading
- 导出完成后显示结果弹窗：
  - 文件名
  - 导出范围
  - 记录数

### 导出历史

页面内新增一个轻量的 `最近导出` 入口，展示：

- 导出时间
- 项目名称
- 导出范围
- 发起人
- 文件名
- 再次下载

## 后端设计

建议新增独立服务：`ProjectDetailExportService`

职责：

- 组装项目导出 payload
- 生成 workbook
- 记录导出历史

### 服务边界

`CostStatisticsService`

- 负责提供成本数据与聚合

`ProjectDetailExportService`

- 负责把项目成本数据整理为导出上下文
- 负责 workbook 构建
- 负责文件名、sheet、格式

### 推荐接口

#### `POST /api/cost-statistics/project-exports`

请求体：

- `project_name`
- `month_scope`
- `start_month`
- `end_month`
- `include_oa_details`
- `include_invoice_details`
- `include_exception_rows`
- `include_ignored_rows`
- `include_expense_content_summary`
- `sort_by`

返回：

- `export_id`
- `file_name`
- `summary`

#### `GET /api/cost-statistics/project-exports/{export_id}`

返回导出任务元数据与状态。

#### `GET /api/cost-statistics/project-exports/{export_id}/download`

返回生成后的 workbook。

## 数据口径

项目明细导出必须显式区分：

- 正常成本明细
- 已处理异常
- 已忽略
- 未唯一匹配 OA 成本上下文

其中：

- 正常成本进入 `项目汇总 / 按费用类型汇总 / 流水明细`
- 异常与忽略不混进正常汇总，单独进入 `异常与未闭环`

## 文件样式要求

- 所有 sheet 冻结首行
- 所有表开启筛选
- 金额列统一两位小数
- 表头加粗
- 说明页保留导出时间与口径
- 工作表列宽按内容预设
- 文件名使用项目名称和范围

## 安全与性能

- 单次导出仅允许一个项目
- 限制自定义时间范围跨度，防止导出过大
- 导出任务需记录操作人
- 如后续数据量上升，可切异步任务模式
