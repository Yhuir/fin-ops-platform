# 关联台三栏局部搜索 / 筛选 / 排序设计

日期：2026-04-08

## 目标

在 `已配对` 与 `未配对` 两个区域内，为 `OA / 银行流水 / 进销项发票` 三栏增加：

- 栏级局部搜索
- 列级多选筛选
- 栏级时间排序

并保持三栏仍然按“候选组”对齐显示，不退化成各栏各自独立列表。

## 核心交互原则

### 1. 当前栏驱动、整组联动

无论是搜索还是筛选，生效规则都一致：

- 当前栏只显示命中的项
- 其他两栏显示这些命中项所属候选组里的全部相关项

这意味着它不是简单的“单栏筛表”，而是“单栏驱动、整组三栏联动”。

### 2. 每个区域一次只开一个搜索框

`已配对` 和 `未配对` 各自独立维护 UI 状态。

在同一个区域内：

- 只允许一个栏级搜索框处于打开状态
- 切到别栏时，前一个搜索框自动关闭

### 3. 列筛选是多选，不是单选

每个列筛选菜单都需要支持：

- 多选
- 全选
- 清空选择

### 4. 排序按组，不按行

`银行流水` 和 `进销项发票` 支持时间升降序切换。

排序不是只改当前栏单独顺序，而是：

- 先为每个候选组计算当前栏的排序键
- 再按该键重排整组

这样三栏仍然能保持对齐。

## 范围拆解

### A. 栏标题工具

每个 pane header 右侧新增：

- 搜索 icon 按钮
- 该栏支持排序时，再加时间排序按钮

### B. 列筛选工具

每个 column header 右侧新增一个下拉三角按钮。

点击后弹出当前列的候选值菜单：

- 多选值
- 全选
- 清空

### C. 组过滤引擎

后端无需新增接口，第一阶段直接在前端基于已加载的 grouped data 做派生：

- 搜索命中集
- 筛选命中集
- 排序后的 group ids

最终生成 `displayGroups`

## 各栏字段调整

### OA

移除：

- `金额`
- `申请事由`

保留适合做筛选的字段，例如：

- 申请人
- 项目名称
- 对方户名

### 银行流水

移除：

- `备注`

新增：

- `按时间升序/降序` 按钮

适合筛选的字段示例：

- 对方户名
- 收入/支出
- 支付账户
- 和发票OA关联情况

### 进销项发票

移除：

- `金额/税率/税额`
- `价税合计`

在 `开票日期` 后增加：

- `按时间升序/降序` 按钮

适合筛选的字段示例：

- 销方名称/识别号
- 购买方名称/识别号
- 开票日期

## 前端状态模型建议

每个 zone 一份状态：

- `activeSearchPaneId`
- `searchQueryByPane`
- `filtersByPaneAndColumn`
- `sortByPane`

其中：

- `paired`
- `open`

两套状态互不影响。

## 过滤顺序

建议统一为：

1. 当前栏搜索
2. 当前栏列筛选
3. 当前栏排序
4. 生成命中 group 集
5. 用这些 group 生成三栏联动显示

## 性能要求

- 不允许因为加搜索/筛选把三栏切换重新变卡
- 只在当前 zone 的前端派生数据上做计算
- 派生结果必须 `useMemo`
- 不在键盘输入时触发整页重新加载
