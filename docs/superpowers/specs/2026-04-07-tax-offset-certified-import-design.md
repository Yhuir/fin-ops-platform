# 税金抵扣已认证发票真实导入设计

日期：2026-04-07

## 目标

将税金抵扣页面中的 `已认证发票导入` 从当前占位态升级为真实可用能力，使页面中的：

- `已认证结果`
- `进项票认证计划` 锁灰状态
- 税金试算

都基于真实导入的认证结果，而不是后端样例数据。

## 当前问题

当前实现存在两层明显缺口：

1. 前端导入弹窗仅支持选文件，不做上传、解析、匹配、刷新。
2. 后端税金抵扣数据仍来自 `TaxOffsetService._month_data` 中的写死样例。

所以当前页面中看到的“2 条已认证结果”，并不是导入出来的真实结果，而是样例数据。

- [2026年2月 进项认证结果  用途确认信息.xlsx](/Users/yu/Desktop/fin-ops-platform/fixtures/测试数据/2026年2月%20进项认证结果%20%20用途确认信息.xlsx)

只处理其中的 `发票` sheet。

## 模板字段口径

第一阶段需要识别的核心字段：

- `数电发票号码`
- `发票代码`
- `发票号码`
- `开票日期`
- `销售方纳税人识别号`
- `销售方纳税人名称`
- `金额`
- `税额`
- `有效抵扣税额`
- `勾选状态`
- `发票状态`
- `勾选时间`
- `发票风险等级`

其中真正进入“已认证结果”的记录，至少应满足：

- `勾选状态` 表示已勾选 / 已认证
- `发票状态` 可用于正常抵扣

## 业务目标

### 1. 已认证结果来源

税金抵扣页中的 `已认证结果` 必须来自：

- 已认证发票导入后的持久化记录

不再直接由写死样例驱动。

### 2. 匹配结果分类

导入后的已认证记录需要分成三类：

1. `matched_in_plan`
   - 能匹配到当月 `进项票认证计划`
2. `certified_outside_plan`
   - 已认证，但不在当前计划中
3. `unmatched_imported_record`
   - 模板记录异常、字段不足、无法形成稳定认证记录

第一阶段页面中必须展示前两类；第三类可先作为导入反馈返回，不必常驻主页面。

### 3. 进项计划回写规则

如果某条已认证记录匹配到了计划票：

- 该计划票状态改为 `已认证`
- 行置灰
- checkbox disabled
- 自动纳入试算

如果某条已认证记录未进入计划：

- 进入右侧抽屉 `已认证但未进入计划`
- 自动纳入试算
- 不强塞回计划主表

### 4. 试算规则

试算继续遵循：

`已认证全部进项票 + 当前勾选的未认证计划票`

已认证票不允许用户通过取消勾选排除。

## 匹配规则

第一阶段匹配优先级：

1. `数电发票号码`
2. `发票代码 + 发票号码`
3. `销售方纳税人识别号 / 销售方纳税人名称 + 开票日期 + 税额`

说明：

- 第一阶段不做模糊 OCR 匹配
- 不做跨月计划智能迁移
- 不做人工冲突解决台账

## 持久化建议

新增独立已认证导入数据，不直接塞进税金页面样例：

- 导入批次
- 导入文件
- 已认证发票明细

最少需要支持：

- 按月份查询已认证发票
- 按发票唯一键去重
- 撤回 / 重复导入时的幂等更新

## 接口建议

第一阶段至少新增：

- `POST /api/tax-offset/certified-import/preview`
- `POST /api/tax-offset/certified-import/confirm`
- `GET /api/tax-offset/certified-imports?month=YYYY-MM`

同时调整：

- `GET /api/tax-offset`
- `POST /api/tax-offset/calculate`

让其读取真实已认证结果，而不是样例。

## 前端交互建议

### 已认证发票导入弹窗

弹窗需要支持：

- 选择多个 Excel
- 预览识别结果
- 展示将被纳入的认证记录数量
- 展示无法识别 / 无法匹配的记录数量
- 确认导入

### 导入完成后

页面应立即刷新：

- 摘要卡
- 试算结果
- 进项票认证计划锁灰状态
- 右侧 `已认证结果` 抽屉

## 本轮不做

- 税局侧真实提交认证
- 税局侧真实撤销认证
- 海关缴款书 / 代扣代缴 / 农产品加计扣除 sheet
- 历史版本对比
- 高级冲突处理页面

## 验收标准

- 已认证导入不再是纯占位弹窗
- 导入后税金抵扣页不再依赖写死 `certified_items`
- 与计划匹配成功的进项票显示灰色且不可勾选
- `已认证但未进入计划` 出现在右侧抽屉
- 试算自动包含全部已认证票
- 使用给定 2026-01 / 2026-02 模板时，系统能稳定读取 `发票` sheet 并形成已认证结果
