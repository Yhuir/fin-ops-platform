# 成本统计页面设计

日期：2026-04-01

## 目标

在现有 `关联台` 与 `税金抵扣` 之外，新增一个独立的 `成本统计` 页面，用于基于**银行流水 + 已关联 OA 字段**做项目和费用视角统计，并支持逐层下钻到具体流水与导出。

## 背景

当前系统已经具备：

- OA 数据同步 / 实时读取
- 银行流水导入与确认
- 关联台内的 OA-流水-发票关系处理
- 项目归集底座

但还缺一个财务人员真正能拿来查“某个月某个项目发生了哪些成本”的页面。

用户要的不是：

- 新的关联弹窗
- 毛利分析
- BI 大屏

而是一个清晰、可追溯、可导出的成本统计页面。

## 范围

本轮覆盖：

- 新增顶部导航按钮 `成本统计`
- 新增独立页面，不使用弹窗
- 按月份统计成本
- 按项目查看成本
- 逐层下钻到具体银行流水
- 导出当前视图

本轮不做：

- 复杂成本分摊
- 跨项目分摊
- 毛利 / 利润口径
- 跨月份趋势分析
- 独立数据仓库

## 核心原则

### 1. 统计以流水为准

成本统计的核心数据源是**银行流水**，不是 OA 单据，也不是发票。

原因：

- 真正反映资金支出的是银行流水
- OA 提供的是补充业务语义
- 发票不应替代成本支付事实

### 2. 流水必须能取到 OA 成本字段

进入成本统计的前提，是这条支出流水能稳定取得：

- 项目名称
- 费用类型
- 费用内容

如果拿不到，不纳入正式成本统计。

### 3. 页面以内联下钻为主

主流程应在同一页面内完成：

- 月份汇总
- 项目明细
- 具体流水

通过面包屑和回退按钮切换，不用弹窗承载统计主流程。

## 数据口径

### 纳入统计

- 支出类银行流水
- 已确认关联到 OA，或已经有稳定 OA 映射字段
- 存在 `project_name + expense_type + expense_content`

### 排除统计

- 收入流水
- 未确认关联流水
- 仅发票数据
- 仅 OA 数据
- 已忽略记录
- 缺失 OA 成本字段的记录

## 页面结构

### 顶部工具区

- 年月选择
- `按月份统计`
- `按项目统计`
- 导出按钮

### 主工作区

#### 层级 1：月份汇总

字段：

- 项目名称
- 费用类型
- 金额
- 费用内容

推荐主键：

- `month + project_name + expense_type + expense_content`

#### 层级 2：项目明细

字段：

- 时间
- 费用类型
- 金额
- 费用内容

#### 层级 3：具体流水

字段：

- 银行流水完整信息
- 对应 OA 项目名称
- 对应 OA 费用类型
- 对应 OA 费用内容

## 导出

导出必须跟随当前层级：

- 月份汇总层导出月份汇总
- 项目明细层导出当前项目明细
- 流水层导出当前流水清单或当前单条流水详情

推荐：

- 优先 `xlsx`
- 可补 `csv`

## 后端设计

建议新增 `CostStatisticsService`。

### 读模型

- `CostStatisticsSummaryRow`
- `CostStatisticsProjectRow`
- `CostStatisticsTransactionRow`

### API

- `GET /api/cost-statistics?month=YYYY-MM`
- `GET /api/cost-statistics/projects/{project_name}?month=YYYY-MM`
- `GET /api/cost-statistics/transactions/{transaction_id}`
- `GET /api/cost-statistics/export?...`

## 前端设计

### 路由

- 新增 `/cost-statistics`

### 状态

- 共享月份上下文
- 页面内维护：
  - 当前视图层级
  - 当前项目
  - 当前 transaction
  - 当前导出上下文

### 交互

- 点月份汇总行进入项目明细
- 点项目明细行进入流水层
- 面包屑可返回上一级

## 成功标准

- 用户能从顶部进入 `成本统计`
- 指定月份下能看到 `项目名称 / 费用类型 / 金额 / 费用内容`
- 能点击项目一路下钻到具体流水
- 导出内容与当前页面视图一致
