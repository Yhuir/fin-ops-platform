# 仓库布局约定

这份文档只约束仓库落盘位置和目录边界，不涉及业务逻辑、运行逻辑或接口行为。

## 顶层目录职责

- `backend/`：Python 后端代码、后端依赖说明、后端运行文档。
- `web/`：React 前端工程、原型页面、前端运行文档。
- `tests/`：自动化测试。
- `docs/`：业务、架构、开发、计划和设计文档。
- `fixtures/`：本地可创建的手工验收文件目录；自动化测试不依赖真实导入文件，目录内容不入库。
- `deploy/`：部署资产、环境模板、反向代理配置、SQL 模板。
- `prompts/`：按阶段拆分的开发 prompt。
- `scripts/`：开发辅助脚本和非业务脚本。

## 落盘规则

- 仓库根目录只保留高信号文件和一级目录，不直接放 Excel、PDF、ZIP、截图或临时导出物。
- 业务需求原文放到 `docs/product/`。
- 跨系统方案、部署形态、总体架构说明放到 `docs/architecture/`。
- 面向实现阶段的接口、规则、测试、开发说明放到 `docs/dev/`。
- 设计稿、实施计划、执行拆解统一放到 `docs/superpowers/`。
- 手工核对样例、银行流水原文件、发票导出、专项票据样本可临时放到本地 `fixtures/`，但真实导入文件不入库。
- 部署说明和环境模板跟部署资产放在同一目录下，不散落到根目录或 `docs/dev/`。

## Fixtures 约定

- 自动化测试需要 Excel 导入输入时，在测试内生成最小 mock 文件，不读取 `fixtures/` 里的真实文件。
- 只用于手工核对的大文件，可放在本地 `fixtures/` 目录，不要回流到仓库顶层或提交到仓库。
- 本地样例优先复用已有子目录：
  - `fixtures/测试数据/`
  - `fixtures/测试用银行流水下载/`
  - `fixtures/银行流水*/`
  - `fixtures/进发票信息导出*/`
  - `fixtures/发票信息导出1-3月/`
  - `fixtures/过路费发票/`

## 文档约定

- 每个文档密集目录优先维护一个 `README.md` 作为索引，而不是把说明堆到根目录 README。
- 新文档命名优先体现用途和范围；设计与计划文档继续沿用现有日期前缀。
- 目录调整时先补索引，再移动内容，避免出现“文件已挪走但入口还在旧位置”的状态。

## 本地产物约定

- 本地产物、缓存和构建输出不入库，包括但不限于：
  - `.DS_Store`
  - `__pycache__/`
  - `*.pyc`
  - `node_modules/`
  - `dist/`
  - `.runtime/`
  - `.superpowers/`
- 如需临时导出、实验文件或调试快照，优先放到已忽略目录，不要占用业务目录。
